# Performance Learning Loop

Goal-aligned learning system that makes the agent measurably better over time. After each post matures, the agent evaluates what worked, why, and whether it moved the needle on user-defined goals. Learnings accumulate into a playbook that the OODA Orient phase consults when generating future content.

## Platform-Aware Evaluation Windows

Different platforms have different engagement curves. Posts are evaluated twice: a first eval captures initial performance, a final eval captures full lifecycle. The delta between them is itself a signal ("this post had legs").

| Platform | First Eval | Final Eval | Rationale |
|----------|-----------|------------|-----------|
| X / Threads / Mastodon | 4h | 24h | Fast-moving feeds, engagement peaks early |
| LinkedIn | 24h | 72h | Algorithm resurfaces content over days |
| Instagram / TikTok / Pinterest | 24h | 7d | Visual content has long tail discovery |
| YouTube | 48h | 14d | Search-driven, slow build |
| Reddit / Bluesky | 4h | 48h | Community-driven, variable virality |
| Blog (Medium, Dev.to, etc.) | 48h | 14d | SEO and share-driven, very long tail |

Evaluations are triggered by Temporal scheduled workflows. Each post gets two evaluation tasks scheduled at publish time based on its platform.

## Signals

The agent learns from three layers of data:

### Platform Metrics (pulled from APIs)
- Impressions / reach
- Likes / reactions
- Comments / replies
- Shares / reposts / retweets
- Clicks (link clicks, profile clicks)
- Saves / bookmarks
- Video views / watch time (where applicable)

### Goal Attribution
Each post is scored against the user's active goals — not just vanity engagement metrics.

Example goal: "Grow LinkedIn to 10K followers"
- Post A: 200 likes, 15 comments, +47 followers → high goal impact
- Post B: 500 likes, 30 comments, +3 followers → high engagement, low goal impact

The playbook learns that Post A's pattern (not Post B's) is what to replicate for this goal.

Attribution precision varies by platform:
- **Direct attribution**: Some platforms expose per-post follower/click data (e.g., LinkedIn post analytics)
- **Delta tracking**: For platforms that don't, track the goal metric (e.g., follower count) before and after the post's evaluation window. Attribute the delta proportionally if multiple posts overlap.

### Computed Scores
- **Engagement rate**: total interactions / impressions
- **Goal impact score**: change in goal metric attributed to this post (normalized 0-1)
- **Combined score**: weighted blend of engagement rate + goal impact, with goal impact weighted higher (default 60/40)

## Post Attribution Tags

Every post is auto-tagged with structured attributes at evaluation time (not at post time — tags describe what the post actually is, not what was intended). Tags are generated by the LLM analyzing the post content.

| Tag | Values |
|-----|--------|
| topic | AI, product updates, culture, industry news, personal story, etc. |
| format | question, story, list, thread, hot take, tutorial, announcement |
| tone | casual, professional, humorous, provocative, inspirational |
| length | short (<100 chars), medium (100-500), long (>500) |
| timeOfDay | morning (6-11), midday (11-14), afternoon (14-18), evening (18-23) |
| dayOfWeek | weekday, weekend |
| hasMedia | image, video, link, carousel, text-only |
| hookType | question, statistic, bold claim, story opener, contrarian take |

Tags enable structured queries: "What's the average engagement rate for question-format posts on LinkedIn posted on weekday mornings?"

## Playbook: Dual-Layer Learning

### Layer 1: Structured Tags

Queryable performance data stored per tag combination. Powers:
- Dashboard charts (engagement by topic, best time to post, format performance)
- Filtering (show me my top 10 posts by goal impact on LinkedIn)
- Statistical correlations ("questions outperform statements 2.3x on LinkedIn for follower growth")

### Layer 2: LLM-Generated Insights

After each final evaluation, the LLM analyzes the post's performance in context of the full playbook and generates a natural-language insight. Insights are:
- Deduplicated (don't repeat what's already known)
- Timestamped (recent insights weighted higher)
- Platform-specific
- Goal-aware ("this worked for follower growth" vs "this worked for engagement")

Example insight: "Your audience on X responds best to short, provocative questions posted mid-morning on weekdays. Long threads underperform unless they tell a personal story."

Insights are stored as an append-only log with a rolling window. Old insights that contradict newer data are marked as superseded. The Orient phase reads the top N most recent, relevant insights.

### Insight Generation Trigger

Insights are not generated after every single post evaluation. Instead:
- After every final evaluation, update the structured tags
- Generate a new insight when: a post is a significant outlier (top or bottom 10% by combined score), OR weekly on a schedule (batch analysis of the week's posts)

This keeps LLM costs proportional to learning value, not post volume.

## Learning Mode (User-Configurable)

Users choose how the playbook influences content generation, with transparent token cost implications. Configurable per platform.

| Mode | What It Does | Extra Token Cost |
|------|-------------|-----------------|
| Lite | Top insights injected into system prompt | ~200 tokens/generation |
| Standard | Top 3-5 performing posts included as few-shot examples | ~800 tokens/generation |
| Full | Insights + few-shot examples | ~1000 tokens/generation |

Default: Standard. Token cost estimates shown in settings UI with a brief explanation of the quality tradeoff.

## User-Facing Intelligence

### Weekly Insight Reports

Generated every Monday (configurable), surfaced in the dashboard:
- Top 3 insights learned this week
- Best and worst performing posts with analysis
- Goal progress with attribution ("You gained 230 followers this week. Your top contributor was [post] with +47.")
- Recommendation for next week's content strategy

### Real-Time Nudges

Before content generation, the agent checks the playbook and surfaces actionable suggestions:
- "Based on your playbook, I'd suggest a short question post for LinkedIn right now. Your last 3 long-form posts underperformed your average by 40%."
- "It's Tuesday morning — your best performing time slot on X. Consider a provocative hot take."
- "You haven't posted about [high-performing topic] in 2 weeks."

Nudges appear in the dashboard and are included in the Orient phase reasoning.

## Data Model

### PostEvaluation

```
PostEvaluation {
  id              String   @id
  orgId           String
  postId          String   // links to existing Post model
  platform        String
  evaluationPass  Int      // 1 = first eval, 2 = final eval

  // Raw metrics
  impressions     Int?
  likes           Int?
  comments        Int?
  shares          Int?
  clicks          Int?
  saves           Int?
  videoViews      Int?

  // Computed
  engagementRate  Float
  goalImpact      Json     // { goalId: string, delta: number, score: float }[]
  combinedScore   Float

  // Tags (generated by LLM at eval time)
  tags            Json     // { topic, format, tone, length, timeOfDay, dayOfWeek, hasMedia, hookType }

  evaluatedAt     DateTime
}
```

### PlaybookInsight

```
PlaybookInsight {
  id            String   @id
  orgId         String
  platform      String
  goalId        String?  // null = general insight, set = goal-specific
  insight       String   // natural language
  confidence    Float    // how much data supports this
  basedOnPosts  String[] // post IDs that informed this insight
  supersededBy  String?  // ID of newer insight that contradicts this
  createdAt     DateTime
}
```

### LearningConfig

```
LearningConfig {
  orgId         String
  platform      String   // or "default" for all platforms
  mode          Enum     // LITE, STANDARD, FULL
  updatedAt     DateTime
}
```

## Integration with Existing Systems

- **Brain OODA Loop**: The Orient phase reads playbook insights and nudges. The Decide phase factors in tag-based performance data when choosing content strategy. The Record phase triggers evaluation scheduling.
- **Content Pipeline**: Before generating content, checks learning mode and injects insights/examples into the LLM prompt accordingly.
- **Goals**: Goal progress tracking consumes PostEvaluation data. Attribution scores feed back into the playbook.
- **Dashboard**: New sections for playbook insights, performance charts, weekly reports, and learning mode configuration.
- **Audit Log**: Evaluation results and insight generation logged for transparency.

## Future: Goal Decomposition (v2)

Once attribution data is flowing, the Orient phase can be upgraded to decompose goals into sub-strategies: "To grow followers, increase post frequency on weekdays, optimize for shares (viral reach), and post more question-format content." This is a reasoning upgrade that consumes v1's attribution data — no data model changes needed.

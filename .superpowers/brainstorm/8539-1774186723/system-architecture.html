Key Data Flows

Autonomous Post Creation

  1. Temporal cron triggers OODA cycle
  2. OBSERVE: gather context + trending
  3. ORIENT: score against goals
  4. DECIDE: pick topics, assign risk
  5. ACT: generate via LLM Adapter (BYOK)
  6. Compliance gate checks content
  7. → Guardian mode: queue for approval
  8. Human approves → publish via Postiz

Agent via MCP

  1. External agent connects via MCP
  2. Calls list_integrations
  3. Calls create_post with content
  4. Compliance engine validates
  5. → Guardian mode: queues for approval
  6. Agent calls get_pending_approvals
  7. Orchestrator agent calls approve_item
  8. Post publishes via Postiz

Mention Response (MVP)

  1. Webhook/poll detects new mention
  2. Engagement engine categorizes it
  3. LLM generates contextual reply
  4. Compliance check on reply
  5. → Guardian mode: queue for approval
  6. Human approves → reply sent

Trust Progression

  1. Start: ALL actions require approval
  2. System tracks approval patterns
  3. After N consistent approvals → suggest
  4. "Auto-approve low-risk posts?"
  5. User opts in per category
  6. Gradually: only novel/risky flagged
  7. Kill switch always available

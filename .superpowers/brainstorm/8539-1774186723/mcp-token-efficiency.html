MCP Token Efficiency Analysis

Is MCP the right protocol for cost-sensitive zero-human companies?

The Token Cost of MCP

Every MCP session starts with tool discovery — the agent's LLM ingests ALL tool definitions into its context window. For 21 tools, that's:

~3,500
tokens for 21 tool schemas
(loaded once per session)
~200
tokens per tool call
(request + response overhead)
~$0.01
per full content cycle
(at Claude Sonnet rates)

Mitigation Strategies

A

Composite Tools (Reduce tool count)

Instead of 21 granular tools, expose 5-6 composite tools with an action parameter:

content.manage({ action: "create" | "list" | "update" | "delete", ... })
engagement.manage({ action: "get_mentions" | "reply" | "triage_dm", ... })
approval.manage({ action: "list_pending" | "approve" | "reject", ... })
brain.manage({ action: "run_cycle" | "set_goal" | "get_progress", ... })
account.info({ action: "list" | "get_rules" | "get_status", ... })

Token savings: ~60% on discovery (5 schemas vs 21). Trade-off: less discoverable, agents must read docs to know sub-actions.

B

Tool Profiles (Right-size per use case)

Expose multiple MCP endpoints with different tool sets:

/mcp/lite    → 6 tools  (create_post, list_posts, approve, reject, list_accounts, run_cycle)
/mcp/full    → 21 tools (everything)
/mcp/approve → 3 tools  (list_pending, approve, reject) — just for human-review agents

Token savings: up to 80% for lite profile. Trade-off: user must choose the right profile. Most will just use /full.

C

Keep Granular + Optimize Descriptions (Recommended)

Keep 21 individual tools but minimize schema verbosity:

  • Short descriptions (1 line max, no examples in schema)
  • Use MCP Resources for context (no token cost until read)
  • Lean parameter schemas (no redundant enums, use references)
  • Document fully in a README resource, not in tool schemas

Token savings: ~40% vs verbose schemas. Trade-off: none — this is just good practice. Granular tools are what agents work best with.

The Real Question: Is MCP Worth It?

Protocol Token Cost Reach Integration Effort
MCP ~3,500 tokens/session Every MCP agent (Claude, GPT, Gemini, Cursor, etc.) Zero — just connect
REST API 0 tokens (direct HTTP) Only agents with custom integrations Custom code per agent
A2A Varies (task-based) Enterprise agents (Salesforce, etc.) Moderate
Paperclip Skill Minimal (skill.md) Only Paperclip companies Low

Verdict: MCP's ~$0.01/cycle token overhead is negligible compared to the content generation cost (~$0.03-0.10/cycle). The reach advantage (any MCP agent connects in seconds) is the core moat. Keep MCP as primary, offer REST API as zero-overhead fallback for cost-sensitive pipelines.
